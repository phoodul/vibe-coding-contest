## 3. 프롬프트 전략

### 3-1. 역할 부여
- CLAUDE.md에 "세계 최고의 Full-stack Senior Engineer이자 UX 디자이너" 페르소나 사전 설정
- 프로젝트별 행동 규칙 정의 (코드 작성 전 implementation_plan.md 승인 필수 등)

### 3-2. 컨텍스트 관리
- 기능별 새 대화(서브에이전트) 시작으로 컨텍스트 오염 방지
- 5개 전문 서브에이전트 역할 분리: Researcher, Architect, Implementer, Reviewer, Designer
- 프로젝트 구조, DB 스키마를 대화 시작 시 자동 주입 (CLAUDE.md)
- implementation_plan.md로 AI와 계획 정렬 후 구현

### 3-3. 단계적 분해
- "계획을 먼저 말해줘, 코드는 아직 짜지 마" → 방향 확인 후 구현
- Sequential Thinking MCP로 복잡한 로직을 단계적으로 분해
- 큰 기능을 작은 단위로 분리하여 순차 요청
