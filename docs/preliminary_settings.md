**원격 제어 — 가장 먼저 해결할 것**

Remote Control은 claude.ai/code와 Claude iOS/Android 앱을 로컬 Claude Code 세션에 연결해줍니다. 핵심은 **tmux 안에서 실행**하는 것입니다. 터미널을 닫거나 claude 프로세스를 중지하면 세션이 종료됩니다. `tmux new -s contest` 안에서 `claude remote-control`을 실행하면, 맥을 닫아도 세션이 유지됩니다. 직장에서 웹으로, 이동 중에 폰으로 동일한 세션을 조종할 수 있습니다. `/config`에서 자동 활성화를 켜두면 매번 설정할 필요도 없습니다.

**서브에이전트 — 역할 분리가 핵심**

서브에이전트는 각각 고유한 컨텍스트 윈도우, 커스텀 시스템 프롬프트, 독립적인 도구 접근 권한을 가지고 실행됩니다. `.claude/agents/` 폴더에 마크다운 파일로 정의하면 됩니다. 위 체크리스트에 있는 5개 역할(researcher, architect, implementer, reviewer, designer)을 미리 만들어두세요. 특히 researcher는 웹 검색만 가능하도록, implementer는 full access로 권한을 차등화하는 게 중요합니다.

**Agent Teams도 고려할 가치가 있습니다.** Agent Teams는 하나의 세션이 팀 리드 역할을 하고 작업을 조율하며, 팀원들은 각자 독립적인 컨텍스트 윈도우에서 작업하고 서로 직접 소통합니다. 활성화는 `export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`로 합니다.

**하네스 엔지니어링 — 주제 공개 전에 끝내야 할 것**

프로젝트 뼈대를 미리 세워두는 게 핵심입니다. Flutter 프로젝트 구조, pubspec.yaml 의존성, analysis_options, 빌드 스크립트를 사전에 준비하면 주제 공개 후 바로 구현에 돌입할 수 있습니다. 멀티 에이전트 세션에서 git 버전 관리 문제를 피하려면 worktree를 구성해야 합니다. 이렇게 하면 각 에이전트가 별도 브랜치에서 충돌 없이 병렬 작업이 가능합니다.
