## 1. 워크플로우 (사용자 명시 흐름)

```
① 수능 킬러 문제 1개 선정
       ↓
② easyedu.ai 에서 폰 노이만 튜터와 함께 풀이
       ↓ (화면 녹화 + 발화 스크립트 동시 저장)
③ 5분 이내 영상 편집 (R1 리포트 카드를 클라이맥스로)
       ↓
④ 유튜브 업로드 (Public 또는 Unlisted)
       ↓
⑤ 영상 링크를 홍보 글에 첨부 → 다채널 게시
```

**핵심 원칙**:
- 모든 홍보 글에 노출되는 URL 은 **`https://easyedu.ai`** (apex, www 없음)
- 모든 영상 썸네일·자막에도 `easyedu.ai` 표기
- 수험생 본인 후기 > 개발자 자체 광고 (베타 1명 후기 활용)

## 3. 영상 시나리오 골조

> 폰 노이만 튜터와 수능 킬러 1문제 함께 풀이 (5분)

```
[0:00 - 0:15] 인트로
  - "수능 킬러 문제, AI 가 풀이 과정을 함께 해주는 도구 — easyedu.ai"
  - 화면: easyedu.ai 랜딩 + 폰 노이만 카드

[0:15 - 0:45] 문제 입력 + Manager 자동 분류
  - 수능 킬러 문제 입력 (예: 2024 수능 30번 / 2025 9월 30번 등)
  - Manager 가 area 자동 판별 → 폰 노이만 라우팅
  - 화면: 자동 분류 메시지 + tutor_name 표시

[0:45 - 3:30] 함께 풀이 (chain trigger 누적 + step decomposition)
  - 폰 노이만이 문제 분석 → 핵심 trigger 1·2·3 제시
  - 학생 막힘 시 풀이 단계별 anchor + nudge
  - 화면: trigger 카드 + KaTeX 수식 + step decomposition 시각화
  - 발화: "여기서 polynomial degree 가 핵심이야 — 왜냐하면 ..."

[3:30 - 4:30] R1 리포트 카드 (클라이맥스)
  - 풀이 완료 → R1 자동 생성
  - "오늘 약했던 부분 / 다음에 마주칠 트리거 / 외워둘 명제"
  - 화면: R1 카드 풀스크린

[4:30 - 5:00] 클로징
  - "easyedu.ai — 무료 베타 모집 중"
  - "콴다는 답을 주지만 우리는 답에 이르는 길을 가르칩니다"
  - 화면: 도메인 + 베타 신청 페이지 QR 코드
```

**준비물**:
- 수능 킬러 문제 1개 (사용자 선정 — 풀이 과정이 가장 깔끔하게 보일 것 우선)
- 깨끗한 시크릿창 + easyedu.ai 첫 접속
- 마이크 (외부 USB 권장 — 발화 품질이 영상 품질의 70%)
- 화면 녹화 도구 (OBS Studio / Mac QuickTime / Windows Game Bar)
