# EasyEdu AI — 종합 AI 활용 리포트

## 1. 프로젝트 개요

- **프로젝트명:** EasyEdu AI
- **한 줄 설명:** AI 활용 차세대 교육 솔루션 — 학생 맞춤 학습 16개 도구 + 교사 업무 자동화
- **프로덕션 URL:** https://vibe-coding-contest.vercel.app
- **개발 기간:** 2026.04.06 ~ 2026.04.13 (8일)
- **참가 형태:** 솔로 개발 (Claude Code CLI 기반 바이브코딩)
- **대상 사용자:** 초등학생~대학생 및 학습을 원하는 모든 사람 + 초·중·고 교사 + 사교육비 부담을 안고 있는 학부모

### 해결하려는 문제

**학생·학부모 측:**
1. 사교육 의존도 — 1:1 튜터링, 영어 회화, 진로 상담은 비용이 높아 모든 가정이 접근할 수 없음
2. 개인화된 학습의 부재 — 교실 수업은 30명 이상에게 동일 속도로 진행되지만 학생마다 수준이 다름
3. 영어 말하기 연습 기회 부족 — 원어민 대화 연습은 접근성과 비용 문제로 대다수에게 닫혀 있음
4. 진로 탐색의 한계 — 5,000개+ 직업군 중 체계적 탐색 어려움
5. 수학 사고과정 훈련 부재 — 기존 앱은 풀이를 보여주지만, "왜 이 전략이어야 하는가"를 깨닫는 훈련 부재

**교사 측:**
1. 학생부 기재 부담 — 세특뿐 아니라 행동특성, 자율활동, 동아리, 진로, 독서 등 10개 영역을 학생 수 × 영역 수만큼 작성. 영역마다 글자수 제한이 다르고 금지어 규칙도 복잡
2. 개인정보 우려 — 학생 이름이 포함된 기재 내용을 AI에 입력하면 개인정보가 LLM에 노출
3. 수업 준비 과부하 — 슬라이드, 워크시트, 테스트, 영상 자료를 각각 만들어야 함
4. 공문서 양식 비효율 — K-에듀파인 규격을 수작업으로 맞추는 데 시간 소요

## 2. AI 활용 전략

### 2-1. 개발 과정: Claude Code CLI 바이브코딩

**개발 도구:** Claude Code CLI (claude-opus-4-6, 1M context window)

선택 이유:
1. **1M 토큰 컨텍스트** — 프로젝트 전체 코드베이스(25개 페이지, 18개 API, 수십 개 컴포넌트)를 한 번에 파악하고 수정 가능
2. **CLI 기반 워크플로우** — 에이전트, 스킬, hooks를 파일 시스템에 정의하여 AI 개발 파이프라인 자동화
3. **MCP 서버 연동** — Supabase, Playwright, Context7, Vercel 등 6개 MCP 서버와 직접 연동

**7개 서브 에이전트 시스템** (`.claude/agents/`):

| 에이전트 | 역할 | 도구 제한 |
|----------|------|-----------|
| Architect | PRD/스키마 설계, implementation_plan.md 작성 | Bash 사용 불가 |
| Designer | UI/UX 피드백, 디자인 토큰 관리 | Playwright 스크린샷 기반 |
| Implementer | 코드 구현 | 전체 도구 사용 |
| Researcher | 기술 문서 조사, 트렌드 분석 | 코드 작성 불가 |
| Reviewer | 코드 리뷰, 보안 점검 | 코드 수정 불가 |
| Debugger | 에러 진단-수정-재검증 루프 | 디버깅 전용 |
| Reporter | AI 빌딩 리포트 자동 생성 | Read/Write 전용 |

**6개 스킬 파일** (`.claude/skills/`):

- `modern-glass-bento` — 글래스모피즘 디자인 시스템

- `git-workflow` — Conventional Commits + GitHub Flow
- `web-testing` — Playwright E2E + Vitest 테스트 패턴
- `explain-code` — 코드 설명 템플릿

**4개 Hooks** (자동화):
- PreToolUse(Bash): 위험 명령어 차단 (`security-guard.sh`)
- PreToolUse(Edit/Write): .env 파일 수정 방지 (`env-guard.sh`)
- PostToolUse(Edit/Write): 코드 수정 후 자동 TypeScript 타입 체크 (`post-edit-lint.sh`)
- Stop: 작업 종료 시 자동 빌드 검증 (`stop-build-check.sh`)

**6개 MCP 서버:**
- Supabase MCP — DB 마이그레이션, SQL 실행
- Playwright MCP — 브라우저 E2E 테스트, 스크린샷
- Context7 MCP — 라이브러리 최신 문서 실시간 조회
- Vercel MCP — 배포 상태, 빌드 로그 조회
- Sequential Thinking MCP — 복잡한 설계 문제 단계적 사고

## 3. 주요 기능별 AI 활용 상세

### 3-1. 소크라테스 AI 튜터 (`/tutor`)

**핵심 전략: Guided Learning + Think-Aloud**

- 답을 직접 주지 않고 질문으로 학생 사고를 이끄는 소크라틱 방식
- 교과목(생활과 윤리, 생명과학I, 언어와 매체)별 커리큘럼 주도 수업
- 단원 → 소단원 → 핵심 개념 순서로 체계적 학습 진행
- `[CONCEPT_DONE: 개념명]` 마커로 완료된 개념 추적, 반복 방지
- Think-Articulate-Reflect 사고 가이드 칩 제공
- RAG 웹 검색: Tavily API로 최신 시사/트렌드 주제도 소크라테스 방식으로 튜터링
- **세션 이어하기**: 로그인 사용자의 학습 진행도를 Supabase에 자동 저장/복원
- **단원 퀴즈**: 학습 완료 후 이해도 확인 퀴즈
- **자료 기반 학습**: PDF/MD 업로드 + URL 추출로 교재 기반 대화
- **학습 요약 정리**: 대화 내용을 구조화된 Markdown 노트로 다운로드

### 3-2. 오일러 수학 튜터 (`/euler-tutor`)

- 오일러(Leonhard Euler) 페르소나의 수학 사고과정 코칭 튜터
- **이미지 입력**: 수학 문제 사진을 업로드하면 AI가 분석
- **Backward reasoning**: 답에서 거꾸로 사고과정을 추적하며 이해 유도
- 풀이를 바로 알려주지 않고, 힌트와 질문으로 수학적 사고력 훈련

### 3-3. 원클릭 수업준비 (`/teacher/lesson-prep`)

- 교사가 주제 또는 파일을 입력하면 **4개 산출물을 동시에 생성**:
  - 수업 슬라이드 개요
  - 학생 워크시트
  - 평가 문항 (테스트)
  - 추천 영상 자료 (Tavily 영상 검색)
- **파일 파싱 지원**: HWP, DOCX, PDF, PPTX 파일 업로드 → 내용 추출 → AI 분석

### 3-4. AI 영어 회화 (`/conversation`)

- **Push-to-Talk 방식**: 스페이스바 누르고 말하기 → Web Speech API STT
- **수준별 시나리오**: 레벨 테스트(10문항) → 카페 주문, 공항, 면접 등
- **모델 분리 전략**:
  - 대화: Haiku 4.5 (2~3문장, 빠른 응답)
  - 리포트: Sonnet 4 (강점/개선점/표현 교정/신규 어휘 분석)
- **음성 흐름**: 사용자 음성 → STT → AI 텍스트 → OpenAI TTS → 오디오 재생
- **에코 방지**: AI 발화 중 마이크 자동 중지 → 완료 후 자동 재개

### 3-5. 인터랙티브 마인드맵 (`/mind-map`)

- 3과목(생활과 윤리, 생명과학I, 언어와 매체) 교과서 PDF 원문 기반
- **정적 구조화 데이터**: AI 생성이 아닌 사전 파싱된 데이터 → 토큰 소모 0
- 360° 방사형 트리 + 부채꼴 레이아웃 + 12색 노드
- 포커스 네비게이션: 노드 클릭 시 하위 트리 확대
- 모바일 반응형: 세로 인덴트 트리 레이아웃 분기

### 3-6. 진로 시뮬레이터 (`/career`)

- MBTI, 관심분야, 적성, 학업 성적 등 12개 항목 입력
- AI가 5,000+ 직업 데이터에서 15~20개 맞춤 추천
- 진로 경로 제시: 현재 → 과목 선택 → 전공 → 직업
- 대학+학과 매칭 + 갭 분석/향상 조언
- 추천 직업별 '더 알아보기' URL 링크

### 3-7. 에베레스트 단어 학습 (`/vocabulary`)

- 18,000개 영어 단어, 10단계 레벨
- 그룹 학습 + 4지선다 퀴즈
- 에베레스트 등반 시각화로 학습 진행도 표현
- **브라우저 내장 SpeechSynthesis**로 발음 재생 → TTS API 비용 0

### 3-8. 영문법 핵심 문장 듣기 (`/grammar-listen`)

- 3레벨 체계: Lv1(A1-A2) + Lv2(B1-B2) + Lv3(C1-C2)
- 각 레벨 100문장 = 총 300문장 반복 청취
- Web Audio API 기반 모바일 호환 재생

### 3-9. 위기 개입 자원 페이지 (`/crisis`)

- 상담 전화, 정부 기관, 디지털 자원, 행동 가이드 종합 안내
- **AI 호출 0**: 정적 데이터로 구성 (위기 상황에서 API 의존성 제거)
- Playwright 기반 링크 검증 완료

### 3-10. 기타 도구

- **AI 도서 추천** (`/books`): 국립중앙도서관 API 연동 + AI 맞춤 요약 + 독서 기록 (Supabase DB)
- **PAPS 체력 분석** (`/paps`): 등급 테이블 매칭(프론트엔드) + AI 맞춤 운동 프로그램
- **진로 탐색기** (`/pathfinder`): AI 기반 진로 경로 탐색
- **음악 감상 도우미** (`/music-review`): AI 음악 감상문 작성 지원
- **직업 체험 시뮬레이터** (`/internship`): 직업 체험 시나리오 AI 생성
- **학생부 기재 도우미** (`/teacher/record`): 2026 기재요령 기반 10개 영역별 글자수 관리 + 금지어 감지 + AI 초안 생성 + 가명처리(개인정보 보호) + AI 맞춤법 검사 + 관찰 기록 스크린샷 첨부
- **공문서 포맷터** (`/teacher/formatter`): 정규식 기반 로컬 분석 + AI 교정
- **공문서 초안 작성기** (`/teacher/generator`): K-에듀파인 양식에 맞는 공문서 초안 AI 생성

### 3-11. 학생부 기재 도우미 상세 (`/teacher/record`)

2026학년도 학교생활기록부 기재요령(208페이지)의 영역별 글자수 제한 데이터를 직접 추출하여 구현:

| 영역 | 한글 기준 최대 글자수 | 바이트 한도 |
|------|---------------------|-----------|
| 과목별 세부능력 및 특기사항 | 500자 | 1,500B |
| 개인별 세부능력 및 특기사항 | 500자 | 1,500B |
| 행동특성 및 종합의견 | 300자 | 900B |
| 자율·자치활동 | 500자 | 1,500B |
| 동아리활동 | 500자 | 1,500B |
| 진로활동 | 500자 | 1,500B |
| 독서활동상황(공통) | 500자 | 1,500B |
| 독서활동상황(과목별) | 250자 | 750B |
| 출결상황 | 500자 | 1,500B |
| 일상생활 활동상황 | 1,000자 | 3,000B |

**바이트 계산 수정:** 기존 EUC-KR(한글 2바이트)에서 2026 기재요령 기준 UTF-8(한글 3바이트, 영문 1바이트, 엔터 1바이트)로 정정.

**가명처리 파이프라인:**
```
교사 입력: "김철수는 토론에서 비례원칙을 분석함"
  ↓ 클라이언트에서 치환
LLM 전송: "학생A는 토론에서 비례원칙을 분석함"
  ↓ LLM 응답
LLM 반환: "학생A는 과잉금지원칙의 세부 요소를..."
  ↓ 클라이언트에서 복원
교사 화면: "김철수는 과잉금지원칙의 세부 요소를..."
```

**추가 기능:** AI 맞춤법 검사(Haiku 기반, 원클릭 교정), 관찰 기록 스크린샷 첨부(Claude Vision), 유사도 분석(학생 간 복붙 방지)

## 4. 사람과 AI의 협업 과정

### "아이디어는 사람, 구현은 AI, 검증은 함께"

이 프로젝트는 솔로 개발자(사람)와 Claude Code(AI)의 긴밀한 협업으로 만들어졌습니다.

**사례 1: 오일러 튜터**
- 사람: "기존 수학 앱은 풀이를 보여주지만, 사고과정을 훈련하는 도구는 없다. 바이브코딩처럼 학생이 전략을 말하고 AI가 계산을 담당하면?"
- AI: Chi(자기설명), Schoenfeld(메타인지), Polya(발견적 학습) 학술 근거 조사 → 5단계 코칭 프롬프트 설계 → 이미지 업로드 + 오일러 아바타 UI 구현
- 함께: Playwright에서 실제 수학 문제로 AI 응답 품질 검증

**사례 2: 학생부 기재 도우미 확장**
- 사람: "세특만 지원하는 건 부족하다. 기재요령 PDF에 10개 영역이 있고, 학생 이름이 AI에 노출되면 안 된다"
- AI: PDF 208페이지에서 영역별 글자수 제한 추출 → 10개 영역 탭 + 동적 바이트 한도 + 가명처리 + 맞춤법 검사 구현
- 사람 추가: "입력란이 좁다. 스크린샷도 첨부하고 싶다" → AI가 textarea 확장 + 이미지 업로드 + Claude Vision 연동
- 함께: 가명처리 전 과정(이름 입력→AI 생성→실명 복원)을 Playwright로 검증

**사례 3: 위기 개입**
- 사람: "교육 앱이라면 학생 안전도 고려해야 한다"
- AI: 정적 데이터로 구성(AI 의존 제거) → Playwright로 외부 링크 8개 수정, 폐쇄 서비스 삭제

**사례 4: AI 영어 회화**
- 사람: "스페이스바를 PTT로, 실시간 회화에는 빠른 모델이 필요하다"
- AI: Haiku(대화) + Sonnet(리포트) 이중 모델 + PTT + 에코 방지 + OpenAI TTS 연동

## 5. UX/UI 혁신

### 5-1. 학습 축하 confetti 애니메이션

- 학습 목표 달성 시 화면 전체에 축하 confetti 효과
- **논문 근거**: micro-interaction이 사용자 세션 시간을 평균 18% 증가시킨다는 UX 연구 기반
- 학습 완료의 성취감을 시각적으로 강화하여 학습 지속 동기 부여

### 5-2. 페이지 전환 애니메이션

- Framer Motion 기반 `template.tsx`에서 전역 페이지 전환 효과
- fade-in + slide-up 조합으로 부드러운 네비게이션 경험

### 5-3. Stats 카운트업 애니메이션

- 랜딩 페이지 통계 수치(학생 수, 도구 수 등)가 뷰포트 진입 시 0에서 목표값까지 스프링 애니메이션
- 스크롤 트리거 기반으로 사용자 시선이 도달했을 때만 재생

### 5-4. 글래스모피즘 + 다크모드

- 반투명 배경 + backdrop-blur + 메시 그라디언트 조합
- 다크모드 기본 적용: `html.dark` 클래스 + CSS 변수 기반 색상 체계
- 노이즈 텍스처 오버레이로 깊이감 표현

### 5-5. PWA 지원

- Service Worker 기반 오프라인 캐싱
- 홈 화면 설치 가능 (`manifest.json`)
- 모바일 반응형 레이아웃 전면 적용

## 6. 토큰 절약 전략

### A. 개발 과정 (Claude Code CLI)

1. **에이전트/스킬 분리**: 7개 에이전트에 역할별 도구만 허용 → 불필요한 탐색 방지
2. **스킬 파일로 반복 프롬프트 제거**: 디자인 규칙, 개발 패턴을 사전 정의
3. **Hooks 자동화**: 코드 수정 후 자동 린트, 작업 종료 시 자동 빌드 체크
4. **implementation_plan.md 기반 작업**: 설계 승인 후 구현 → 잘못된 구현 재작업 방지
5. **Memory 시스템**: 프로젝트 컨텍스트를 메모리 파일로 유지 → 새 세션에서 재설명 불필요
6. **CLAUDE.md 규칙**: "Simplicity First", "Surgical Changes" 등 행동 규칙으로 불필요한 코드 생성 방지

### B. 프로덕트 (사용자)

**B-1. 모델 분리**: 영어 회화(대화)에 Haiku 4.5 → Sonnet 대비 비용 ~1/10

**B-2. AI를 호출하지 않는 기능 설계**:

| 기능 | AI 대신 사용한 방식 | 절약 효과 |
|------|---------------------|----------|
| 영어 단어 발음 (18,000개) | 브라우저 `SpeechSynthesis` | TTS API 비용 0 |
| 마인드맵 (3과목) | 교과서 원문 기반 정적 데이터 | 토큰 소모 0 |
| NEIS 바이트 계산·금지어 감지 | 클라이언트 JavaScript | AI 불필요 |
| 가명처리 (개인정보 보호) | 클라이언트 문자열 치환 | AI 불필요 |
| PAPS 등급 기본 분석 | 프론트엔드 테이블 매칭 | AI 불필요 |
| 위기 개입 자원 페이지 | 정적 데이터 | AI 호출 0 |

**B-3. 로컬 저장으로 재호출 방지**: 단어 학습·문법 듣기·독서 기록을 localStorage/Supabase에 저장

**B-4. 스트리밍 응답**: 모든 AI API에서 `streamText()` 사용 → 사용자가 중간에 떠나면 토큰 생성 자연 중단

**B-5. 시스템 프롬프트 최적화**: 개념 완료 마커(`[CONCEPT_DONE]`)로 반복 질문 방지

## 7. ChatGPT Study Mode 대비 차별화

| 비교 항목 | ChatGPT Study Mode | EasyEdu AI |
|-----------|-------------------|------------|
| 학습 방식 | 범용 AI 대화 | 교과목별 전용 커리큘럼 + 소크라틱 방식 |
| 교과서 연동 | 없음 | 3과목 교과서 원문 마인드맵 + 단원별 학습 |
| 진도 추적 | 대화 기록만 | 개념별 완료 추적 + 세션 이어하기 + 퀴즈 |
| 수학 전용 | 없음 | 오일러 튜터 (이미지 입력 + backward reasoning) |
| 영어 회화 | 텍스트만 | PTT 음성 대화 + TTS + AI 리포트 |
| 진로 시뮬레이터 | 없음 | 5,000+ 직업 매칭 + 대학·학과 추천 |
| 교사 도구 | 없음 | 원클릭 수업준비, 공문서, 생기부 4개 전용 도구 |
| 위기 개입 | 없음 | 상담 자원 종합 페이지 (AI 의존 0) |
| 오프라인 | 웹 전용 | PWA 지원 (오프라인 캐싱) |

**핵심 차별점**: ChatGPT Study Mode는 "똑똑한 AI와 자유 대화"이지만, EasyEdu AI는 "한국 교과과정에 특화된 16개 전문 도구"를 제공합니다. 커리큘럼 주도 학습, 교과서 원문 마인드맵, 교사 전용 도구는 범용 AI에서 구현할 수 없는 영역입니다.
