# AI 활용 리포트 — EduFlow AI

## 1. 프로젝트 개요
- **프로젝트명:** EduFlow AI
- **한 줄 설명:** AI 기반 차세대 교육 솔루션 — 학생 맞춤 학습 + 교사 행정 자동화
- **해결하려는 문제:** 학생의 수동적 학습/진로 불안 + 교사의 반복적 공문서 행정 업무
- **대상 사용자:** 중고등학교 학생 및 교사

---

## 2. 사용한 AI 도구 및 역할 분담

| AI 도구 | 활용 영역 | 선택 이유 |
|---------|----------|----------|
| **Claude Code (Opus 4.6, 1M context)** | 아키텍처 설계, 전체 구현, 디버깅, 배포 | 100만 토큰 컨텍스트로 프로젝트 전체를 한 대화에서 관리 |
| **Claude API (Sonnet)** | 소크라테스 튜터, 진로 매칭, 도서 추천, 공문서 생성 | Vercel AI SDK `streamText`로 실시간 스트리밍 UX |
| **Supabase MCP** | DB 스키마 마이그레이션, RPC 함수 생성 | 자연어 → SQL DDL 자동 변환, 즉시 적용 |

## 3. 프롬프트 전략

### 3-1. 역할 부여 (CLAUDE.md)
- "세계 최고의 Full-stack Senior Engineer이자 UX 디자이너" 페르소나 설정
- 행동 규칙 명시: "코드 수정 전 implementation_plan.md 작성 후 승인", "능동적 의견 제시"
- 코딩 가이드라인: Simplicity First, Surgical Changes, Goal-Driven Execution

### 3-2. 컨텍스트 관리
- **서브에이전트 5종** 역할 분리: Researcher(조사), Architect(설계), Implementer(구현), Reviewer(검증), Designer(UX)
- **Auto Memory 시스템**: 프로젝트 진행 상황, 기술 결정, 피드백을 파일 기반으로 영속 저장 → 대화 간 컨텍스트 유지
- **implementation_plan.md**: AI와 계획을 먼저 정렬한 뒤 구현에 돌입

### 3-3. 단계적 분해
1. "이 기능의 설계안을 먼저 제시해줘" → 방향 확인
2. "진행해줘" → AI가 Task 리스트 생성 → 순차 구현
3. 각 단계마다 빌드 검증 (`next build` 에러 0 확인)
4. 완료 후 커밋 → push → Vercel 배포 → 빌드 로그 확인

## 4. AI로 해결한 핵심 문제

### 문제 1: 소크라테스식 튜터 프롬프트 설계
- **상황:** "답을 직접 주지 않고 질문으로 이끄는 AI 튜터"는 일반 챗봇과 완전히 다른 프롬프트 전략이 필요
- **프롬프트:** "Khanmigo 연구 결과를 참고하여 절대 답을 주지 않는 소크라테스식 튜터 system prompt를 설계해줘. 힌트 3단계 시스템 포함."
- **AI 응답 핵심:** 5가지 행동 규칙(역질문 의무, 힌트 레벨 자동 상승, 정답 도달 시 개념 정리)을 구조화한 system prompt 생성
- **내 판단:** 수용 — 교육학적 근거(Khanmigo 수학 성적 14% 향상)를 바탕으로 한 설계
- **결과:** 교과별(국어/사회/과학) 튜터가 답을 직접 주지 않고 단계적으로 유도하는 UX 구현

### 문제 2: 공문서 번호 동시성 안전 발급
- **상황:** 200명 졸업장 일괄 발급 시 같은 번호가 부여되면 안 됨
- **프롬프트:** "여러 사용자가 동시에 요청해도 중복 번호가 발생하지 않는 atomic 번호 발급 시스템을 설계해줘"
- **AI 응답 핵심:** PostgreSQL `INSERT ON CONFLICT DO UPDATE` 활용한 RPC 함수 제안
- **내 판단:** 수용 — DB 레벨 atomic 연산이 가장 안전한 방법
- **결과:** `generate_doc_number()` RPC 함수로 `EduFlow-0001(2026. 4. 8.)` 형식 번호 발급, 동시 호출에도 중복 없음

### 문제 3: SHA-256 해시 기반 위변조 검증
- **상황:** 발급된 공문서의 원본 여부를 기술적으로 증명할 수 있어야 함
- **프롬프트:** "파일 업로드 시 해시를 생성하고, 수정 시 해시 변경을 추적하며, 나중에 검증할 수 있는 시스템"
- **AI 응답 핵심:** Web Crypto API (외부 의존성 없음) + document_versions 테이블로 전체 이력 추적
- **내 판단:** 수용 — 브라우저 네이티브 API로 추가 패키지 없이 구현 가능
- **결과:** 업로드→수정→승인 전 과정의 해시 이력이 타임라인으로 시각화, 파일 업로드만으로 원본 검증 가능

### 문제 4: K-에듀파인 공문서 양식 규칙 엔진
- **상황:** 한국 공문서에는 날짜(`2026. 4. 8.`), 시간(`15 : 30`), 금액(`금50,000원(금오만원)`) 등 독특한 표기 규칙이 존재
- **프롬프트:** "K-에듀파인 공문서 양식 규칙을 코드 포맷터처럼 자동 교정하는 규칙 엔진을 만들어줘"
- **AI 응답 핵심:** 5가지 규칙(날짜/시간/금액/항목 기호/끝 표시)을 regex 기반으로 구현, `numberToKorean()` 한글 금액 변환 포함
- **내 판단:** 수용 + 보완 — AI가 생성한 regex를 실제 공문서 샘플로 검증
- **결과:** 텍스트 붙여넣기 → 실시간 diff 뷰 → 한 줄씩 또는 일괄 적용하는 "코드 포맷터 같은" UX

### 문제 5: Vercel 보안 취약점 배포 차단 해결
- **상황:** Next.js 15.3.1 CVE-2025-66478 취약점으로 Vercel이 배포 거부
- **프롬프트:** (빌드 로그를 AI가 직접 Vercel MCP로 조회하여 원인 파악)
- **AI 응답 핵심:** 같은 15.3 라인의 패치 버전(15.3.9)으로 업그레이드 제안
- **내 판단:** 수용 — 메이저 업그레이드 대신 패치 버전으로 안전하게 해결
- **결과:** 빌드 로그 조회 → 원인 파악 → 패치 → 재배포 성공까지 3분 이내 해결

## 5. 나 <-> AI 워크플로우 (솔로 개발 사이클)

### 개발 사이클
```
아이디어 토의 → AI가 설계안 제시 → 승인/수정
    |
    v
implementation_plan.md 작성 → Task 리스트 생성
    |
    v
Task 순차 구현 (AI 자율 작업) → 각 단계마다 빌드 검증
    |
    v
커밋 → push → Vercel 자동 배포 → MCP로 배포 상태 확인
    |
    v
다음 기능으로 이동
```

### 효과적이었던 패턴
1. **"설계안 먼저, 코드는 나중"**: 모든 기능에 대해 AI가 먼저 설계안(DB 스키마, 페이지 구조, 파일 목록)을 제시 → 승인 후 구현. 방향 오류로 인한 재작업 0건
2. **MCP 도구 체인**: Supabase MCP(DB 마이그레이션) → 코드 구현 → Vercel MCP(배포 확인)까지 한 대화에서 끊김 없이 진행
3. **Auto Memory**: 이전 대화의 결정 사항, 기술 스택, 진행 상황을 자동 저장 → 새 대화 시작 시 컨텍스트 재구축 불필요
4. **Task 기반 진행 관리**: AI가 스스로 Task 리스트를 생성하고 하나씩 완료 표시 → 빠짐없이 진행

### 한계와 극복 방법
1. **HWP 파일 텍스트 추출 불가**: HWP는 한국 고유 포맷으로 브라우저 파싱이 어려움 → 양식 검토 시 "건너뛰기" 옵션 제공, 해시 검증은 바이너리 기준으로 동작
2. **UI 디자인 감각의 한계**: AI가 생성한 코드는 기능적으로 완전하지만 "감각적 디자인"은 사람의 판단 필요 → Playwright MCP로 스크린샷 확인 후 수동 피드백

## 6. 성과 요약

| 지표 | 수치 |
|------|------|
| 총 개발 시간 | ~24시간 (04.08~04.08, 집중 개발) |
| AI 활용 비율 | ~95% (설계, 구현, 디버깅, 배포 전 과정) |
| AI로 절약한 시간 (추정) | ~80시간 (5개 기능 + 워크플로우를 하루 만에 구현) |
| 총 커밋 수 (version2) | 16개 |
| 구현된 페이지 수 | 21개 라우트 |
| DB 테이블 수 | 11개 (Auth 포함) |
| API 엔드포인트 수 | 7개 |

### AI 없이는 불가능했을 기능
1. **소크라테스 AI 튜터**: 교과별 전문 지식 + 절대 답을 주지 않는 행동 규칙이 결합된 프롬프트 설계
2. **5,000+ 직업 매칭**: 학생 프로필을 분석하여 잘 알려지지 않은 직업까지 추천하는 LLM 추론
3. **공문서 자동 생성**: 자유 텍스트 → 두문-본문-결문 K-에듀파인 공문서 구조 변환

### 핵심 교훈
> AI는 "코딩 도우미"가 아니라 "풀스택 파트너"다. 설계-구현-배포-검증의 전 사이클을 MCP 도구와 함께 한 대화에서 끊김 없이 진행할 수 있고, 중요한 것은 AI에게 "무엇을 만들지"보다 "왜 만드는지"를 정확히 전달하는 것이다.
