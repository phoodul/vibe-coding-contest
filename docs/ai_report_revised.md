## 1. 기획

### 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

**세 가지 사용자 그룹**을 설정했습니다.

**학생(초등~대학생):** 사교육 의존도가 높고, 개인화된 학습이 부재합니다. 1:1 튜터링·영어 회화·진로 상담은 비용이 높아 모든 가정이 접근할 수 없습니다. 수학은 풀이를 보여주는 앱은 많지만, "왜 이 전략이어야 하는가"를 깨닫는 사고과정 훈련이 부재합니다. 영어 말하기 연습은 원어민 접근성과 비용 문제로 대다수에게 닫혀 있습니다.

**학부모:** 연간 수백만 원의 사교육비 부담을 안고 있습니다. 통계청 2025년 기준 월 평균 사교육비는 40만 원 이상입니다.

**교사:** 학생부 기재 부담이 매우 큽니다. 세특뿐 아니라 행동특성, 자율활동, 동아리, 진로, 독서 등 10개 영역을 학생 수 × 영역 수만큼 작성해야 하고, 영역마다 글자수 제한과 금지어 규칙이 다릅니다. 학생 이름이 포함된 기재 내용을 AI에 입력하면 개인정보가 LLM에 노출되는 우려도 있습니다. 수업 준비(슬라이드, 워크시트, 테스트, 영상 자료)를 각각 만들어야 하는 과부하, K-에듀파인 공문서 규격을 수작업으로 맞추는 비효율도 해결해야 할 문제입니다.

### 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

**학생용 12개 + 교사용 4개 = 총 16개 AI 도구**를 하나의 플랫폼에 통합했습니다.

**학생용 핵심 기능:**
1. **소크라테스 AI 튜터** — 답을 주지 않고 질문으로 이끄는 Guided Learning. URL을 입력하면 콘텐츠를 분석해 섹션별로 분류하고, 개요 모드(넓게 빠르게)와 실무 모드(깊게 실전적) 중 선택하여 학습. AI가 5~10가지 학습 항목을 자동 도출해 수업 계획을 세우고, 학생 이해도에 따라 유동적으로 진행합니다.
2. **오일러/가우스 수학 튜터** — 오일러(Claude Sonnet 4.5)와 가우스(GPT-5.1) 듀얼 튜터를 토글로 전환. 10년치 수능 수학 530문항의 풀이를 Supabase에 캐싱하여 기출문제에서 올바른 방향으로 코칭을 보장. Ctrl+V 스크린샷 붙여넣기 → Mathpix/Upstage/Vision 3단계 OCR 파이프라인으로 수식을 정확히 인식합니다.
3. **AI 영어 회화** — Push-to-Talk 음성 대화, 레벨별 시나리오, AI 리포트(강점/교정/어휘 분석).
4. **에베레스트 단어 학습** — 18,000단어를 10구간(Base Camp~Summit)으로 나눠 등반하듯 학습. 퀴즈 합격 시 등산객 캐릭터가 올라가는 3초 애니메이션.
5. **진로 시뮬레이터** — MBTI/적성/흥미 분석 → 5,000+ 직업 매칭 + 대학·학과 추천.

**교사용 핵심 기능:**
1. **원클릭 수업준비** — 주제 입력 → 슬라이드, 워크시트, 평가문항, 추천 영상 4개 산출물 동시 생성.
2. **학생부 기재 도우미** — 2026 기재요령 기반 10개 영역 관리 + 가명처리(학생 이름을 "학생A"로 치환 후 AI 전송, 응답에서 복원) + 금지어 감지 + AI 맞춤법 검사.
3. **공문서 포맷터/초안 작성기** — K-에듀파인 규격 자동 검사 및 공문서 초안 AI 생성.

### 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

- **학생:** Bloom의 2 Sigma 연구에 따르면 1:1 튜터링은 평균 성적 20%p 향상 효과가 있습니다. AI 튜터로 이 효과를 비용 없이 제공합니다.
- **학부모:** 월 40만 원 이상의 사교육비 절감 가능. 1:1 영어 회화, 수학 코칭, 진로 상담을 무료 AI로 대체합니다.
- **교사:** TALIS 2024 기준 교사 행정 업무에 주당 5시간 이상을 소모합니다. 학생부 기재, 수업 준비, 공문서 작성을 AI가 보조하여 이 시간을 수업과 학생 지도에 재투자할 수 있습니다.
- **교육 형평성:** 지역, 소득 수준에 관계없이 모든 학생이 양질의 1:1 교육에 접근 가능합니다.

## 2. AI 활용 전략

### 이 프로젝트에서 사용할 AI 도구(Claude Code, Cursor, Codex 등)와 모델은 무엇이며, 선택한 이유는 무엇인가요?

**개발 도구: Claude Code CLI (claude-opus-4-6, 1M context window)**

선택 이유:
1. **1M 토큰 컨텍스트** — 프로젝트 전체 코드베이스(26개 페이지, 21개 API, 수십 개 컴포넌트)를 한 번에 파악하고 수정 가능. Cursor나 Copilot의 8K~32K 컨텍스트로는 프로젝트 전체를 볼 수 없습니다.
2. **CLI 기반 워크플로우** — 에이전트(.claude/agents/), 스킬(.claude/skills/), hooks를 파일 시스템에 정의하여 AI 개발 파이프라인을 체계화. GUI 도구에 비해 자동화와 재현성이 뛰어납니다.
3. **MCP 서버 연동** — Supabase, Playwright, Context7, Vercel 등 6개 MCP 서버와 직접 연동하여 AI가 DB 마이그레이션, 브라우저 테스트, 배포 확인까지 자율적으로 수행합니다.

**프로덕트에 사용한 AI 모델:**

| 용도 | 모델 | 선택 이유 |
|------|------|-----------|
| 소크라테스 AI 튜터 | Claude Sonnet 4 | 교육적 대화에 필요한 깊은 추론 + 한국어 품질 |
| 오일러 수학 튜터 | Claude Sonnet 4.5 | 수학 추론 최적화, 2025 수능 76.2% 정답률 |
| 가우스 수학 튜터 | OpenAI GPT-5.1 | AIME 94.6%, 수학 벤치마크 최상위, 비용 절반 |
| AI 영어 회화 (대화) | Claude Haiku 4.5 | 응답 속도 3~5배 빠름 (실시간 회화 필수) |
| AI 영어 회화 (리포트) | Claude Sonnet 4 | 대화 전체 분석: 강점/개선점/표현 교정 |
| TTS (음성 합성) | OpenAI gpt-4o-mini-tts | 자연스러운 영어 음성 출력 |
| 웹 검색 (RAG) | Tavily API | 최신 시사 정보 + 기출문제 풀이 검색 |
| 수식 OCR | Mathpix + Upstage | 수학 문제 이미지 → LaTeX 텍스트 변환 |
| 기타 14개 API | Claude Sonnet 4 | 범용 추론 (진로, 도서, 공문서, 학생부 등) |

### 각 AI 도구별 활용 전략을 작성해주세요.

**1. 7개 서브 에이전트 시스템 (`.claude/agents/`)**

역할별로 분리된 에이전트가 각자의 도구 제한 내에서 작업합니다:
- **Architect**: PRD/스키마 설계, implementation_plan.md 작성 (Bash 사용 불가 → 실행 없이 설계만)
- **Implementer**: 코드 구현 (전체 도구 사용)
- **Reviewer**: 코드 리뷰, 보안 점검 (코드 수정 불가 → 검토만)
- **Debugger**: 에러 진단-수정-재검증 루프
- **Designer**: UI/UX 피드백 (Playwright 스크린샷 기반)
- **Researcher**: 기술 문서 조사 (코드 작성 불가)
- **Reporter**: AI 활용 리포트 자동 생성

**2. 6개 스킬 파일 (`.claude/skills/`)**

재사용 가능한 개발 패턴을 스킬로 정의하여 일관성 확보:
- `nextjs-dev`: Next.js 15 + Shadcn/ui + Supabase + Vercel AI SDK 개발 패턴
- `modern-glass-bento`: 글래스모피즘 디자인 시스템 (CSS 변수, 컬러 팔레트)
- `code-review`: 심각도별 코드 리뷰 체크리스트
- `git-workflow`: Conventional Commits + GitHub Flow
- `web-testing`: Playwright E2E + Vitest 테스트 패턴
- `explain-code`: 코드 설명 템플릿

**3. 4개 Hooks (자동화)**

AI의 실수를 사전/사후에 자동으로 방지:
- `security-guard.sh` (PreToolUse/Bash): `rm -rf /`, `DROP TABLE` 등 위험 명령어 차단
- `env-guard.sh` (PreToolUse/Edit): .env 파일 수정 방지 → API 키 유출 예방
- `post-edit-lint.sh` (PostToolUse/Edit): 코드 수정 후 자동 TypeScript 타입 체크
- `stop-build-check.sh` (Stop): 작업 종료 시 자동 빌드 검증

**4. 프로덕트 AI 활용 전략**

- **소크라테스 튜터**: 시스템 프롬프트에 "답을 주지 말고 질문으로 이끌어라"는 핵심 제약을 걸어 ChatGPT Study Mode와 차별화. `[CONCEPT_DONE]` 마커로 학습 진행도를 프론트엔드에서 실시간 추적. URL 자료 기반 학습 시 AI가 내부적으로 5~10가지 학습 항목과 순서를 계획한 뒤 자연스럽게 첫 항목부터 가이드.
- **오일러/가우스 듀얼 튜터**: 동일 시스템 프롬프트로 두 모델(Sonnet/GPT-5.1)을 토글 전환. 기출문제 선택 시 Supabase에 캐싱된 검증 풀이를 시스템 프롬프트에 자동 주입하여 올바른 코칭 방향을 보장.
- **영어 회화 모델 분리**: 대화는 Haiku(빠른 응답), 리포트는 Sonnet(깊은 분석)으로 분리하여 비용과 품질을 동시에 최적화.
- **학생부 가명처리**: 학생 실명을 "학생A/B/C"로 치환한 뒤 AI에 전송, 응답에서 원래 이름으로 복원 → 개인정보 보호와 AI 활용을 양립.
- **기출문제 풀이 DB**: 10년치 수능 수학 530문항의 풀이를 Tavily 웹 검색으로 수집 → Supabase에 캐싱. 기출문제에서 AI가 틀린 방향으로 코칭하는 것을 원천 방지.

**5. 6개 MCP 서버 연동**

| MCP 서버 | 활용 |
|----------|------|
| Supabase | DB 마이그레이션, SQL 실행, 풀이 DB 테이블 생성 |
| Playwright | 브라우저 E2E 테스트, 반응형 스크린샷 검증 |
| Context7 | Next.js, Supabase, Vercel AI SDK 최신 문서 실시간 조회 |
| Vercel | 배포 상태 확인, 빌드 로그 조회 |
| Sequential Thinking | 복잡한 아키텍처 설계 시 단계적 사고 |
| Firebase | 추가 인프라 옵션 |

### 토큰 낭비를 최소화하고 유지보수성 및 재현성을 높이기 위한 전략이 있다면 작성해주세요.

**토큰 절약 전략:**
1. **모델 티어링** — 작업 복잡도에 따라 모델을 분리. 영어 회화 대화는 Haiku($0.25/1M), 분석은 Sonnet($3/1M), 수학은 Sonnet 4.5($3/1M). 모든 작업에 고비용 모델을 사용하지 않습니다.
2. **정적 데이터 활용** — 마인드맵(3과목 교과서)과 위기 개입 페이지는 AI 호출 0. 사전 파싱된 정적 데이터로 토큰 소모 없이 동작.
3. **기출문제 원문 내장** — 10년치 432문항 텍스트를 JSON 파일(188KB)로 번들에 포함. 매번 OCR을 돌리지 않습니다.
4. **풀이 캐싱** — Tavily로 한 번 검색한 풀이는 Supabase에 저장. 같은 문제를 다시 검색하지 않습니다.
5. **브라우저 내장 API** — 영어 단어 발음은 SpeechSynthesis(무료), 회화 STT는 Web Speech API(무료)를 사용하여 외부 API 호출 최소화.
6. **Upstage/Mathpix 우선순위** — 수식 OCR은 Mathpix(정확) → Upstage(범용) → Vision(비쌈) 순서로 시도. 비용 대비 정확도 최적화.

**유지보수성 전략:**
1. **CLAUDE.md** — 프로젝트 규칙(코딩 스타일, 제약 사항, 커밋 컨벤션)을 파일에 명시. 새 대화를 시작해도 AI가 동일한 품질로 작업.
2. **Memory 시스템** — 20개+ 메모리 파일로 프로젝트 맥락, 사용자 선호도, 피드백을 대화 간 유지.
3. **Hooks 자동화** — 빌드 검증, 타입 체크, 보안 검사가 자동으로 실행되어 사람이 깜빡해도 품질 보장.
4. **Conventional Commits** — `feat:`, `fix:`, `docs:` 접두사로 커밋 히스토리의 가독성과 추적성 확보.

**재현성 전략:**
1. **.env.example** — 필요한 모든 환경변수(11개)를 placeholder와 함께 문서화.
2. **Supabase 마이그레이션** — DB 스키마 변경을 SQL 마이그레이션으로 관리. 새 환경에서 동일 DB 구조 재현 가능.
3. **Skills 파일** — 개발 패턴을 스킬로 문서화하여 동일 품질의 코드를 반복적으로 생성.
