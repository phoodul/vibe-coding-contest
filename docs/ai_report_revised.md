## 1. 기획

### 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

**세 가지 사용자 그룹**을 설정했습니다.

**학생(초등~대학생):**
1. 개인화된 학습의 부재 — 교실 수업은 30명 이상에게 동일 속도로 진행되지만 학생마다 수준이 다름. 1:1 튜터링은 비용이 높아 대부분의 학생이 접근할 수 없음
2. 수학 사고과정 훈련 부재 — 기존 앱은 풀이를 보여주지만, "왜 이 전략이어야 하는가"를 깨닫는 사고과정 훈련이 부재
3. 영어 말하기 연습 기회 부족 — 원어민과의 대화 연습은 접근성과 비용 문제로 대다수에게 닫혀 있음
4. 진로 탐색의 한계 — 5,000개+ 직업군 중 체계적으로 탐색할 수 있는 도구가 없음
5. 학습 동기 유지 어려움 — 혼자 공부할 때 목표 설정, 진행도 시각화, 성취감 부여가 부족

**학부모:**
1. 사교육비 부담 — 통계청 2025년 기준 월 평균 사교육비 40만 원 이상. 1:1 영어 회화, 수학 코칭, 진로 상담까지 포함하면 가계에 큰 부담
2. 학습 관리의 한계 — 자녀의 학습 수준과 진행 상황을 객관적으로 파악하기 어려움
3. 교육 격차 우려 — 지역, 소득 수준에 따라 교육 기회가 불평등하게 분배됨

**교사:**
1. 학생부 기재 부담 — 세특뿐 아니라 행동특성, 자율활동, 동아리, 진로, 독서 등 10개 영역을 학생 수 × 영역 수만큼 작성. 영역마다 글자수 제한이 다르고 금지어 규칙도 복잡
2. 개인정보 우려 — 학생 이름이 포함된 기재 내용을 AI에 입력하면 개인정보가 LLM에 노출
3. 수업 준비 과부하 — 슬라이드, 워크시트, 테스트, 영상 자료를 각각 만들어야 함
4. 공문서 양식 비효율 — K-에듀파인 규격을 수작업으로 맞추는 데 시간 소요

### 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

**학생용 12개 + 교사용 4개 = 총 16개 AI 도구**를 하나의 플랫폼에 통합했습니다.

**학생용 핵심 기능:**

1. **소크라테스 AI 튜터** (`/tutor`) — 답을 주지 않고 질문으로 이끄는 Guided Learning. 교과목(생활과 윤리, 생명과학I, 언어와 매체, 영어, 바이브코딩 30단원)별 커리큘럼 주도 수업. `[CONCEPT_DONE]` 마커로 완료된 개념 추적. URL을 입력하면 HTML 헤딩 기반으로 섹션을 자동 분리하고, 학생이 원하는 섹션을 선택한 뒤 개요 모드(30K자, 넓게)와 실무 모드(5K자, 깊게) 중 선택하여 학습. AI가 5~10가지 학습 항목을 자동 도출해 수업 계획을 세우고, 학생 이해도에 따라 유동적으로 진행. 세션 이어하기(Supabase), 단원 퀴즈, 학습 요약 다운로드 제공.

2. **오일러/가우스 수학 튜터** (`/euler-tutor`) — 오일러(Claude Sonnet 4.5)와 가우스(GPT-5.1) 듀얼 튜터를 토글로 전환. 10년치 수능 수학 512문항 원문을 Mathpix로 파싱하여 내장. 530문항 풀이를 Tavily 웹 검색으로 수집 → Supabase에 캐싱하여, 기출문제 선택 시 풀이를 시스템 프롬프트에 자동 주입 → 올바른 코칭 방향 보장. KaTeX 수식 렌더링으로 LaTeX를 교과서 수준 수식으로 표시. 정답을 직접 알려주지 않고 보기를 다시 보여주며 학생이 선택하도록 유도. 기본 계산 단계(정적분 대입, 인수분해 등)는 학생에게 먼저 질문. Ctrl+V 스크린샷 붙여넣기 → Mathpix/Upstage/Vision 3단계 OCR 파이프라인.

3. **AI 영어 회화** (`/conversation`) — Push-to-Talk(스페이스바) 음성 대화. 레벨 테스트(10문항) → 카페 주문, 공항, 면접 등 시나리오. 대화는 Haiku 4.5(빠른 응답), 리포트는 Sonnet 4(강점/교정/어휘/Native 표현 제안/학습 방향 통합). 10턴 이상 대화 후 리포트 생성. 음성 흐름: 사용자 음성 → Web Speech API STT → AI → OpenAI TTS → 오디오 재생. 에코 방지, 자막 기본 OFF.

4. **에베레스트 단어 학습** (`/vocabulary`) — 18,000단어를 10구간 11포인트(Base Camp + Camp 1~9 + Summit)로 나눠 등반하듯 학습. 각 구간 1,800단어 = 45묶음 × 40단어, 90% 이상 합격 시 통과. 퀴즈 합격 시 등산객 캐릭터(모자+배낭+스틱 SVG)가 올라가는 3초 클라이밍 오버레이 애니메이션. 정상 도달 시 만세 포즈 + 깃발. 브라우저 SpeechSynthesis로 발음 재생(TTS 비용 0).

5. **진로 시뮬레이터** (`/career`) — MBTI/적성/흥미 등 12개 항목 → 5,000+ 직업 매칭 + 대학·학과 추천. ReactMarkdown 렌더링 + URL 클릭 가능. 인라인 편집 + 다운로드 가능.

6. **기타 학생 도구** — 내 길 내비(진로 로드맵), 맞춤 도서 추천(국립중앙도서관 API, 제목/저자 분리 검색, 도서만 필터, 최신순 정렬), 인터랙티브 마인드맵(3과목 교과서 정적 데이터, 360° 방사형 트리), 영문법 핵심 300문장(3레벨 반복 청취), PAPS AI 코치(종목별 설명 + AI 맞춤 운동), 현장실습 일지(당일 날짜 자동), 음악 감상문 코치

**교사용 핵심 기능:**

1. **원클릭 수업준비** (`/teacher/lesson-prep`) — 주제 입력 → 슬라이드(25~30장, 총 8,000자+, 장당 200~300자), 워크시트(A4 2장, 3개 활동), 평가문항(10문항, 난이도 배분), YouTube 영상 3개를 동시 생성. 2Slides API 연동으로 디자인된 PPT 파일 다운로드 가능. PDF/HWP/DOCX 파일 업로드 시 Upstage Document Parse로 파싱.

2. **학생부 기재 도우미** (`/teacher/record`) — 2026 기재요령(208페이지) 기반 10개 영역별 글자수 관리(UTF-8 바이트 계산). 가명처리(실명→"학생A" 치환 → LLM 전송 → 응답에서 복원). 금지어 감지. AI 맞춤법 검사(Haiku). 관찰 기록 스크린샷 첨부(Claude Vision). temperature 0.6(사실 기반 + 표현 다양성).

3. **공문서 포맷터** (`/teacher/formatter`) — K-에듀파인 규격 정규식 기반 로컬 분석(날짜, 시간, 금액, 항목 기호, 붙임, 수신자 등) + AI 교정. 영문 혼용 감지 규칙.

4. **공문서 초안 작성기** (`/teacher/generator`) — K-에듀파인 양식에 맞는 공문서 초안 AI 생성.

**공통 기능:**
- **EditableResult**: 6개 페이지(진로, 도서, PAPS, 현장실습, 음악감상, 내길내비)에서 AI 결과를 인라인 편집 + Markdown 다운로드
- **대시보드 학생/교사 탭 전환**: 로그인 없이도 양쪽 기능 접근 가능
- **위기 개입 자원**: 상담 전화·기관·행동 가이드(AI 호출 0)
- **PWA**: 홈 화면 설치 + 오프라인 캐싱

### 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

- **학생:** Bloom의 2 Sigma 연구에 따르면 1:1 튜터링은 평균 성적 20%p 향상 효과가 있습니다. 소크라테스 AI 튜터와 오일러 수학 튜터로 이 효과를 비용 없이 제공합니다. 에베레스트 등반 시각화와 confetti 애니메이션으로 학습 동기를 유지합니다.
- **학부모:** 월 40만 원 이상의 사교육비 절감 가능. 1:1 영어 회화(OpenAI TTS), 수학 코칭(오일러+가우스), 진로 상담(5,000+ 직업)을 무료 AI로 대체합니다.
- **교사:** TALIS 2024 기준 교사 행정 업무에 주당 5시간 이상을 소모합니다. 학생부 기재(10개 영역 + 가명처리), 수업 준비(원클릭 4개 산출물 + PPT 다운로드), 공문서 작성을 AI가 보조하여 이 시간을 수업과 학생 지도에 재투자할 수 있습니다.
- **교육 형평성:** 지역, 소득 수준에 관계없이 모든 학생이 양질의 1:1 교육에 접근 가능합니다.

## 2. AI 활용 전략

### 이 프로젝트에서 사용할 AI 도구(Claude Code, Cursor, Codex 등)와 모델은 무엇이며, 선택한 이유는 무엇인가요?

**개발 도구: Claude Code CLI (claude-opus-4-6, 1M context window)**

**Claude Code CLI 선택 이유:**
1. **VS Code 통합** — 익숙한 VS Code 환경에서 터미널을 열어 바로 작업 가능. 별도 IDE를 배우거나 전환할 필요 없이, 기존 개발 환경 그대로 AI와 협업
2. **Remote Control로 언제 어디서든 작업** — 근무 중 점심시간이나 이동 시간에도 모바일로 Claude Code에 접속하여 작업 지시 가능. 8일간의 대회 기간 동안 시간을 최대한 활용할 수 있었음
3. **1M 토큰 컨텍스트** — 프로젝트 전체 코드베이스(26개 페이지, 21개 API, 수십 개 컴포넌트)를 한 번에 파악하고 수정 가능. 총 250개+ 커밋을 8일간 생산. Cursor나 Copilot의 8K~32K 컨텍스트로는 불가능한 수준
4. **CLI 기반 워크플로우** — 에이전트(.claude/agents/), 스킬(.claude/skills/), hooks를 파일 시스템에 정의하여 AI 개발 파이프라인 자동화. GUI 도구에 비해 자동화와 재현성이 뛰어남
5. **MCP 서버 연동** — Supabase, Playwright, Context7, Vercel, Sequential Thinking, Firebase 6개 MCP 서버와 직접 연동하여 AI가 DB 마이그레이션, 브라우저 테스트, 배포 확인까지 자율적으로 수행

**Opus 4.6 모델 선택 이유:**
- 단순히 MVP나 프로토타입을 보여주려는 것이 아니라, **실제 교육 현장에서 바로 활용 가능한 수준의 앱**을 만들고 싶었음
- 16개 도구 각각이 교사와 학생이 실제로 사용할 수 있는 품질이어야 했기 때문에, 코드 품질·아키텍처 설계·프롬프트 엔지니어링 모두에서 최고 성능의 모델이 필요했음
- Sonnet으로는 복잡한 시스템 설계(멀티 에이전트, OCR 파이프라인, 풀이 DB 아키텍처, KaTeX 수식 렌더링 등)를 한 번에 정확하게 구현하기 어려웠고, Opus의 깊은 추론 능력이 프로덕션 품질 달성의 핵심이었음

**프로덕트에 사용한 AI 모델:**

| 용도 | 모델 | 선택 이유 |
|------|------|-----------|
| 소크라테스 AI 튜터 | Claude Sonnet 4 | 교육적 대화에 필요한 깊은 추론 + 한국어 품질 |
| 오일러 수학 튜터 | Claude Sonnet 4.5 | 수학 추론 최적화, 2025 수능 76.2% 정답률 |
| 가우스 수학 튜터 | OpenAI GPT-5.1 | AIME 94.6%, 수학 벤치마크 최상위, 비용 절반 |
| AI 영어 회화 (대화) | Claude Haiku 4.5 | 응답 속도 3~5배 빠름 (실시간 회화 필수) |
| AI 영어 회화 (리포트) | Claude Sonnet 4 | 대화 전체 분석: 강점/개선점/Native 표현 교정 |
| TTS (음성 합성) | OpenAI gpt-4o-mini-tts | 자연스러운 영어 음성 출력 |
| 웹 검색 (RAG) | Tavily API | 최신 시사 정보 + 기출문제 풀이 검색 |
| 수식 OCR | Mathpix + Upstage | 수학 문제 이미지 → LaTeX 텍스트 변환 |
| 슬라이드 생성 | 2Slides API | 텍스트 → 디자인된 PPT 파일 생성 |
| 맞춤법 검사 | Claude Haiku 4.5 | 학생부 기재 내용 문법 교정 (빠르고 저렴) |
| 기타 14개 API | Claude Sonnet 4 | 범용 추론 (진로, 도서, 공문서, 학생부, PAPS 등) |

### 각 AI 도구별 활용 전략을 작성해주세요.

**1. 8개 서브 에이전트 시스템 (`.claude/agents/`)**

역할별로 분리된 에이전트가 각자의 도구 제한 내에서 작업합니다:

| 에이전트 | 역할 | 도구 제한 |
|----------|------|-----------|
| Architect | 아키텍처 설계, PRD 작성, DB 스키마 설계 | 코드 실행 불가 — 설계만 |
| Designer | UI/UX 피드백, 디자인 토큰 관리 | Playwright 스크린샷 기반 |
| Implementer | implementation_plan.md에 따라 기능 코드 구현 | 전체 도구 사용 |
| Improver | 코드 품질 개선, 리팩토링, 성능 최적화 | 기존 코드 분석 후 개선 |
| Researcher | 기술 문서 조사, 트렌드 분석, 라이브러리 비교 | 코드 작성 불가 |
| Reviewer | 코드 리뷰, 보안 점검, OWASP 취약점 검사 | 코드 수정 불가 — 검토만 |
| Debugger | 빌드/런타임 에러 자동 탐지 → 수정 → 재검증 루프 | 터미널 + Playwright |
| Reporter | AI 빌딩 리포트 자동 생성 | Read/Write 전용 |

**2. 6개 스킬 파일 (`.claude/skills/`)**

재사용 가능한 개발 패턴을 스킬로 정의하여 일관성 확보:
- `nextjs-dev`: Next.js 15 + Shadcn/ui + Supabase + Vercel AI SDK 개발 패턴
- `modern-glass-bento`: 글래스모피즘 디자인 시스템 (CSS 변수, 컬러 팔레트)
- `code-review`: 심각도별 코드 리뷰 체크리스트
- `git-workflow`: Conventional Commits + GitHub Flow
- `web-testing`: Playwright E2E + Vitest 테스트 패턴

**4. 프로덕트 AI 활용 전략 (핵심)**

- **소크라테스 튜터 — ChatGPT Study Mode와의 차별화**: 시스템 프롬프트에 "답을 주지 말고 질문으로 이끌어라"는 핵심 제약. `[CONCEPT_DONE]` 마커로 학습 진행도를 프론트엔드에서 실시간 추적. URL 자료 기반 학습 시 AI가 내부적으로 5~10가지 학습 항목과 순서를 계획한 뒤 자연스럽게 첫 항목부터 가이드. ChatGPT Study Mode가 "똑똑한 AI와 자유 대화"라면, 소크라테스 튜터는 "한국 교과과정에 특화된 커리큘럼 주도 학습"입니다.

- **오일러/가우스 듀얼 튜터 — 기출문제 올바른 코칭 보장**: 동일 시스템 프롬프트로 두 모델(Sonnet 4.5/GPT-5.1)을 토글 전환. 10년치 수능 530문항의 풀이를 Tavily 웹 검색 → Supabase 캐싱. 기출문제 선택 시 정답+풀이를 시스템 프롬프트에 자동 주입하여 AI가 틀린 방향으로 코칭하는 것을 원천 방지. KaTeX 렌더링으로 수식을 교과서처럼 시각적으로 표시. 정답 미노출 원칙: 보기를 다시 보여주며 학생 스스로 선택하도록 유도.

- **영어 회화 모델 분리**: 대화는 Haiku(응답 3~5배 빠름, 비용 1/10), 리포트는 Sonnet(깊은 분석). 에코 방지: AI 발화 중 마이크 자동 중지. 10턴 이상 대화해야 리포트 생성 가능(충분한 데이터 확보).

- **학생부 가명처리 파이프라인**: 교사 입력 "김철수는 토론에서 비례원칙을 분석함" → 클라이언트에서 "학생A"로 치환 → LLM 전송 → 응답에서 "학생A"를 "김철수"로 복원. 개인정보가 LLM에 노출되지 않으면서 AI 활용 가능.

- **기출문제 풀이 DB 아키텍처**: 530문항 풀이를 사전 수집하여 Supabase에 캐싱. 학생이 기출문제를 선택하면 API가 DB에서 해당 문제의 정답+풀이를 조회하여 시스템 프롬프트에 주입. 학생 메시지에는 정답이 포함되지 않음(서버측에서만 처리).

- **EditableResult 공통 컴포넌트**: 6개 페이지에 AI 결과 인라인 편집 + Markdown 다운로드 기능 적용. ReactMarkdown으로 구조화된 렌더링 보장.

**6. 사람과 AI의 협업 과정 — "아이디어는 사람, 구현은 AI, 검증은 함께"**

- **오일러 튜터**: 사람이 "기존 수학 앱은 풀이만 보여준다. 사고과정을 훈련하자"고 방향 제시 → AI가 학술 근거(Chi 자기설명, Schoenfeld 메타인지, Polya 발견적 학습) 조사 + 5단계 코칭 프롬프트 설계 + OCR 파이프라인 구현 → Playwright로 실제 수학 문제 AI 응답 품질 검증
- **학생부 확장**: 사람이 "세특만 지원하면 부족하다. 10개 영역 + 가명처리가 필요하다" 요구 → AI가 기재요령 208페이지에서 영역별 글자수 추출 + 가명처리 파이프라인 구현 → Playwright로 가명처리 전 과정 검증
- **위기 개입**: 사람이 "교육 앱이면 학생 안전도 고려해야 한다" → AI가 정적 데이터로 구성(API 의존 제거) → Playwright로 외부 링크 검증
