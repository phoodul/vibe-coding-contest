## 1. 마일스톤 & timing

```
[Phase 0 — 지금~2주]   베타 5명 무료 + 리뷰 수집  (결제 미도입)
       ↓
[Phase 0 종료]          베타 KPI 검증 (별점 평균 / 코멘트 / 구매 의향)
       ↓
[Phase 1 시작 ~2주]     사업자 주소 이전 (자택→부산 임대) + 통신판매업 신고
       ↓
[Phase 1 +1주]          토스페이먼츠 가입 + 심사 + 라이브 키
       ↓
[Phase 1 +2~3주]        앱 SDK 통합 + 약관 4종 공개 + 베타 → 유료 전환
```

**핵심 원칙**: 결제 도입은 베타 KPI 검증 후. 지금 통합하면 베타 사용자가 결제 페이지 보고 부담 → 진정성 손상.

## 2. 결제 인프라 — 토스페이먼츠 1순위

### 2-1. PG 선택 근거

| PG | 추천 | 이유 |
|---|---|---|
| **토스페이먼츠** | ⭐⭐⭐ Phase 1 단독 | 한국 1위. 정기결제 안정. 카드/토스/카카오/네이버페이 통합. Next.js SDK 완성도 높음 |

### 2-2. 토스페이먼츠 가입 절차

| 서류/정보 | 비고 |

| 사업자 통장 (정산 계좌) | 사업자등록증 + 신분증으로 은행 방문 |
| 대표자 신분증 | |
| 사업장 사진 | 홈오피스도 OK (책상·노트북 사진) |

1. **tosspayments.com** → "가입하기" → 사업자 정보 입력

4. 승인 → **테스트 키** 즉시 발급
5. 정산 계좌 인증 (1원 입금) → **라이브 키** 발급

#### 수수료 (한국 표준)

| **네이버페이** | 2.5~2.9% |

#### 정산

## 3. 사업자 주소 이전 (자택 → 부산 소상공인 임대)

### 3-1. 결정 (사용자, 2026-05-03)

- **자택 → 부산시 소상공인 임대로 이전 후 PG 가입**
- 사업자등록증 주소 변경 + 통신판매업 신고증 + 토스페이먼츠 가입을 한 번에 정렬

### 3-2. 부산 소상공인 임대 옵션

| 유형 | 비용 (월) | 특징 |
|---|---|---|
| **공유오피스** | ₩50,000~150,000 | 책상 1자리, 회의실 공유, 사업자등록 가능 |
| **부산 창업지원센터 입주** | 무료~저렴 | 부산경제진흥원 / 부산창조경제혁신센터, 심사 필요 |
| **비상주 사무실 (주소만 임대)** | ₩30,000~70,000 | 사업자등록 주소만 빌림, 실제 근무는 자택 |
| **소호 사무실 (1인 독립실)** | ₩200,000~500,000 | 본격 사무실 |

추천 단지: 부산 센텀 / 해운대 / 동래 / 사상 — 교통 + 인프라 + IT 클러스터.

### 3-3. 전환 절차

1. **임대 계약** → 임대차계약서 보관
2. **세무서** 방문 또는 홈택스 → 사업자등록증 주소 변경 (당일~1주)
3. **정부24** → 통신판매업 신고 (신규, 변경된 주소 기준)
4. **사업자 통장** 개설 (사업자등록증으로)
5. **토스페이먼츠 가입** (위 §2-2)
6. 약관·개인정보처리방침에 사업자 주소·통신판매업 신고번호 반영

## 4. 세금 (한국 개인사업자 SaaS)

### 4-1. 부가가치세 (VAT 10%)

- **일반과세자 등록 권장** (간이과세자보다 매입세액 공제 + 세금계산서 발급 가능 → SaaS 유리)
- **분기별 신고** (1·4·7·10월) + **확정신고** (1·7월)
- **B2C 가격 표시 = VAT 포함 의무** (전자상거래법)
  - ₩29,000 (VAT 포함) → 매출 ₩26,364 + VAT ₩2,636
  - ₩49,000 → ₩44,545 + ₩4,455
  - ₩99,000 → ₩90,000 + ₩9,000
- 결제 PG 수수료(2.9%) 매입세액 공제

### 4-2. 종합소득세 (매년 5월)

- 첫 해 매출 8,000만원 미만 → **추계신고** 가능 (간단)
- 8,000만원 초과 → **복식부기 의무** (장부)
- 1억원 초과 + 순이익 1억원 → **법인 전환 검토**

### 4-3. 회계 SW + 세무사 timing

| 매출 구간 | 권장 |
|---|---|
| ~월 100만원 | 직접 처리 + **삼쩜삼** (종소세 자동) |
| 월 100만~500만원 | **캐시노트 / 자비스 / 알라쿰바** (월 1~3만원) |
| 월 500만원~ | **세무사** (월 10~15만원) — 부가세·종소세 위탁 |

### 4-4. 영수증 의무

- 카드결제 → 자동 현금영수증 (PG 처리)
- B2B 결제 → 세금계산서 별도 발행 (홈택스 또는 회계 SW)

## 5. 환불 (한국 전자상거래법)

### 5-1. 법정 환불 정책 (의무)

디지털 콘텐츠/구독:
- **결제일 ≤ 7일 + 서비스 미이용** → 전액 환불 의무
- **사용한 경우** → 사용분 일할 차감 후 환불 (약관 명시 시)
- **정기결제** → 다음 결제일 전 해지 가능 + 일할 환불 또는 다음 갱신 안 함

### 5-2. 이용약관 필수 조항

1. **환불 가능 조건** (결제일 7일 이내 + 미이용)
2. **부분 환불 산정 기준** (월 결제 후 사용 일수 / 전체 일수 × 결제액 차감)
3. **정기결제 해지 방법** + 해지 후 잔여 기간 처리
4. **환불 절차** (신청 경로 + 처리 영업일 + 카드 입금 일수 3~5영업일)

### 5-3. PG (토스페이먼츠) 환불 기능

- 관리자 대시보드 → 거래 선택 → **"환불" 버튼**
- 전체 / 부분 환불 모두 지원
- 카드 환불: 3~5영업일 (카드사 처리)
- **API 자동화**: 앱에서 사용자 신청 → 관리자 승인 → 토스 API 호출 → 자동 환불

### 5-4. 앱 내 환불 UI 설계 (Phase 1 +3주)

```
사용자 흐름:
  [My Page] → [결제 내역] → 거래 선택 → [환불 신청]
    → 사유 입력 (드롭다운: 미이용 / 기능 부족 / 가격 / 기타)
    → 7일 이내 자동 승인 / 7일 초과 관리자 검토
    → 토스 API 환불 호출
    → 카드사 입금 (3~5영업일)
    → 사용자 알림 (이메일 + 앱 토스트)

관리자 흐름:
  [/admin/payments] → 거래 검색 → 환불 사유 검토 → 승인/거절
    → 거절 시 이유 회신
```

### 5-5. 분쟁 예방 원칙

- **약관 명확** + **환불 정책 투명** → 결제 전환율 ↑ (사용자가 환불 가능을 알면 더 결제함)
- **빠른 응대 (24시간 이내)** + **친절한 톤** → 분쟁 거의 없음
- 분쟁 발생 시: 1372 소비자상담센터 → 분쟁조정 → 공정위 (전자상거래법 위반)

## 6. 약관 4종 (Phase 1 +2~3주 작성)

결제 도입 시점에 사이트에 공개해야 할 페이지:

| 경로 | 내용 |
|---|---|
| `/terms` | **이용약관** — 환불 정책 + 정기결제 해지 + 서비스 범위 |
| `/privacy` | **개인정보처리방침** — 수집 항목 / 보관 기간 / 위탁 / 권리 |
| `/refund` | **환불 정책** (별도 페이지로 명확히) — §5 기준 |
| `/business-info` | **사업자 정보** — 상호 / 대표자 / 사업장 주소 / 사업자등록번호 / 통신판매업 신고번호 / 연락처 / 호스팅 정보 |

→ Phase 1 시점에 자율 작성 예정 (15차 세션 이후 별도 task).
