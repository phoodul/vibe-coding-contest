## Harness Engineering의 핵심

1.  Claude.md 파일 및 mcp, skills.md 파일 설정
2. 훅, 미들웨어 등을 설정
3. 테스트, 린터, 타입 체커
4. 컨테이너 환경
5. 상태관리

다만 각각의 **목적**을 이해하는 게 중요합니다. 5가지는 사실 3가지 역할로 묶입니다.

---

## 하네스의 3가지 역할

### 🧭 방향 설정 (AI가 뭘 해야 하는지)
- `CLAUDE.md`, `AGENTS.md`, `skills.md`, MCP 설정
- AI가 작업 시작 전에 읽는 **설계도**

### 🔒 품질 강제 (AI가 틀렸을 때 막는 것)
- pre-commit hook, 미들웨어
- 린터, 타입 체커, 테스트
- AI가 규칙을 어기면 **물리적으로 차단**

### 🔄 상태 유지 (AI가 맥락을 잃지 않도록)
- 컨테이너 환경 (재현 가능한 실행 환경)
- `progress.md`, 세션 간 상태 관리
- AI가 대화가 끊겨도 **어디까지 했는지 기억**

### 피드백 루프

- 피드백 루프는 "실패 결과를 AI에게 다시 돌려주는 것

```
AI 코드 작성 → 테스트 실패 → 실패 로그를 AI에게 전달 → AI 재시도
```

이 순환이 없으면 AI가 실패해도 그냥 멈춥니다.

---

## 한 문장 요약

> AI에게 **방향을 주고** → 틀리면 **막고** → 맥락을 **유지시키고** → 실패를 **다시 돌려주는** 것이 하네스 엔지니어링입니다.
